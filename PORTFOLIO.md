# Lumi Line Reserve MVP

個人サロン向けに、LINE から予約、予約確認、変更・キャンセルまで行える予約導線を実装した実動 MVP です。  
営業用モックではなく、Webhook、LIFF、Supabase、管理画面まで含めて実際に動くことを重視して構築しました。

## プロジェクト概要

- サービス名: `Lumi Line Reserve MVP`
- 想定顧客: まつげ、眉毛、ネイル系の一人営業サロン
- 解決したい課題:
  - 施術中に電話に出られず予約機会を逃しやすい
  - 外部予約サイトだけに依存すると手数料や運用コストがかかる
  - LINE から予約、確認、変更・キャンセルまで完結する導線が欲しい

## 実装したこと

### ユーザー向け

- LINE 内から予約フォームへ遷移できる導線
- 空き枠だけを表示する予約フォーム
- 予約番号の自動発行
- 予約確認画面
- 変更申請、キャンセル申請画面
- アクセス、営業時間、注意事項、Google マップ導線

### 管理者向け

- 管理画面ログイン
- 予約一覧、予約詳細、ステータス更新
- 管理メモ保存
- メニュー管理
- 休業枠管理
- 店舗設定管理

### LINE 連携

- Messaging API Webhook 受信
- 署名検証
- LIFF を使った予約導線
- リッチメニュー運用を前提にした LINE 導線の整備

## 技術スタック

- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase PostgreSQL
- LINE Messaging API
- LIFF SDK
- Vercel

## 構成

```text
LINE公式アカウント
  ↓
LIFF / リッチメニュー導線
  ↓
Next.js App Router
  ↓
API Routes
  ↓
Supabase PostgreSQL
  ↓
管理画面 / 予約運用
```

## こだわったポイント

### 1. 実動MVPとして成立させたこと

見た目だけのモックではなく、予約保存、管理画面反映、変更申請、Webhook 受信まで通した構成にしました。

### 2. 二重予約を防ぐ構成

フロントの空き枠表示だけに頼らず、DB 側でも重複予約を防ぐ関数を用意し、保存時の競合に備えました。

### 3. 一人サロン向けの運用に寄せたこと

複雑なスタッフ管理や多店舗対応は入れず、必要な予約運用に集中させました。  
管理画面も実務重視で、一覧性と判断しやすさを優先しています。

### 4. LINE 導線の現実解を選んだこと

LINE 内導線は挙動差が出やすいため、まずは確実に運用できる構成を優先しました。  
現在は機能別 LIFF を使った導線で安定動作を優先しつつ、将来的な 1 LIFF 統合も見据えています。
